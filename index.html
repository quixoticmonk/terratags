Terratags

A powerful tool for validating tags on AWS resources in Terraform configurations.

Ensure consistent tagging across your infrastructure and maintain compliance with organizational tagging policies.

Get Started View Examples

Key Features

Tag Validation

Validates required tags on AWS resources to ensure compliance with your organization's tagging policies.

Default Tags Support

Supports AWS provider default_tags and tracks tag inheritance from provider to resources.

Module-Level Tags

Validates tags at the module level for consistent tagging across module resources.

Exemption Support

Supports exemptions for specific resources that may not require certain tags.

HTML Reports

Generates detailed HTML reports of tag compliance across your infrastructure.

Auto-Remediation

Provides suggestions for fixing non-compliant resources automatically.

Quick Start

Installation

go install github.com/terratags/terratags@latest

Basic Usage

terratags -config config.yaml -dir ./terraform

Check out the documentation for more detailed information on configuration and usage.

Why Use Terratags?

Consistent tagging is crucial for:

  • Cost allocation and tracking
  • Resource ownership identification
  • Environment classification
  • Security and compliance requirements
  • Automation and resource management

Terratags helps enforce these tagging standards across your infrastructure, ensuring that all resources are properly tagged according to your organization's policies.
